# Phase 3: Community Resource

**Priority:** Low (future)
**Goal:** Share the tool with other KDM players.
**Prerequisite:** Phase 2 (the tool is polished and the data is trustworthy)

## Milestones

### 3.1 — Deployment

- Host the web app publicly (options: VPS, Railway, Fly.io, Vercel + serverless API)
- Evaluate whether SQLite scales or if a migration to PostgreSQL is needed
- Set up CI/CD for automated builds and deploys

### 3.2 — User Contributions

- Allow users to suggest corrections (moderated, not direct edits)
- Submission review workflow (approve/reject/edit)
- Attribution for contributors

### 3.3 — Data Export

- Export gear data as JSON, CSV
- API access for third-party tools and companion apps
- Versioned data snapshots so downstream consumers can pin to a known state

### 3.4 — Legal Considerations

- Review Kingdom Death IP / fan content policies
- Ensure card images comply with fair use (reference/educational context)
- Add appropriate attribution and disclaimers

## Done When

- The tool is live and accessible to the KDM community
- Other players can use it and suggest corrections
- Data can be consumed by other tools and apps
